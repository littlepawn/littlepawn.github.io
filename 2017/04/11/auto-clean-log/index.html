    • ubuntu磁盘空间占满处理办法

    1. 情况描述

      ssh登陆主服务器之后，发现使用tab进行命令自动补全的时候，终端提示：
      Cannot create temp file for here document: No space left on device
      意思是空间不足，用如下命令查找大文件
      df -h and find / -size +1G
      发现是日志文件占空间太大，删除之

    2. 编写定时脚本删除日志

      • 删除文件命令：

        说明：将/opt/soft/log/目录下所有30天前带”.log”的文件删除。具体参数说明如下：
        find：linux的查找命令，用户查找指定条件的文件；
        /opt/soft/log/：想要进行清理的任意目录；
        -mtime：标准语句写法；
        +30：查找30天前的文件，这里用数字代表天数；
        "*.log"：希望查找的数据类型，”.jpg”表示查找扩展名为jpg的所有文件，”“表示查找所有文件，这个可以灵活运用，举一反三；
        -exec：固定写法；
        rm -rf：强制删除文件，包括目录；
        {} \; ：固定写法，一对大括号+空格++;

      • 创建shell脚本

        新建一个可执行文件auto-del-30-days-ago-log.sh，并分配可运行权限

      • 计划任务

        这里的设置是每天凌晨0点10分执行auto-del-30-days-ago-log.sh文件进行数据清理任务了。
